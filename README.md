## 📖 О проекте

Этот репозиторий — **агрегатор публичных бесплатных VPN-конфигураций**, работающих в России для обхода блокировок РКН (Роскомнадзора).

Каждые **6 часов** скрипт автоматически:
- 📥 Собирает конфиги из открытых источников
- 🔍 Убирает дубликаты
- ✅ Проверяет TCP-доступность каждого сервера и пытается установить соединение
- 📤 Публикует три файла с актуальными результатами

Никакой регистрации. Никаких аккаунтов. Скопируйте ссылку и вставьте в клиент.

Нужна стабильность без нагрузки на общий пул — см. раздел [«Стабильный VPN — личный ключ»](#-стабильный-vpn--личный-ключ).

## 📂 Три файла — в чём разница?

Это главный вопрос новичков. Вот простое объяснение:

### `all_configs.txt` — все найденные конфиги

Содержит **все** конфиги, найденные во всех источниках, после удаления дубликатов. Часть из них может не работать прямо сейчас — сервера бывают перегружены, отключены или заблокированы.

➡️ **Когда брать:** есть время самому протестировать, или стандартный файл не помогает.

```
https://raw.githubusercontent.com/aviamastersgh/vpn-free-russia/main/all_configs.txt
```

### `verified_configs.txt` — только рабочие ✅ (рекомендуется)

Из всего пула конфигов отбираются только те, к которым удалось установить **реальное соединение** в момент проверки.

➡️ **Когда брать:** в большинстве случаев — это лучший выбор для начала.

```
https://raw.githubusercontent.com/aviamastersgh/vpn-free-russia/main/verified_configs.txt
```

### `ru_configs.txt` — обход белых списков ⚠️

Для тех, у кого **мобильный интернет с жёсткими ограничениями** (Мегафон, МТС, Билайн, Т2, Yota и др. в режиме белых списков). Содержит конфиги с российскими IP-адресами (CIDR-диапазоны), которые не блокируются даже при максимальных ограничениях оператора.

➡️ **Когда брать:** `verified_configs.txt` не помогает, интернет в режиме белых списков, открывается только Яндекс, ВКонтакте, Госуслуги.

```
https://raw.githubusercontent.com/aviamastersgh/vpn-free-russia/main/ru_configs.txt
```

## 🤔 Как определить, какой файл нужен мне?

**Шаг 1.** Проверьте: открывается ли у вас `google.com` без VPN?

- **Да** → обычный интернет (чёрные списки). Берите `verified_configs.txt`.
- **Нет, но Яндекс/ВК/Госуслуги работают** → белые списки. Берите `ru_configs.txt`.
- **Ничего не открывается** → нет интернета вообще, конфиги не помогут.

**Шаг 2.** Добавьте ссылку как подписку в клиент (инструкции ниже).

**Шаг 3.** Нажмите «Тест задержки» / «Real Delay» — выберите сервер с зелёным числом.

## 📱 Как добавить конфиги в клиент — пошаговые инструкции

### Android — v2rayNG

1. Скачайте [v2rayNG](https://github.com/2dust/v2rayNG/releases) с GitHub
2. Откройте приложение → ☰ (меню) → **«Группа подписки»**
3. **+** → вставьте ссылку на подписку → сохраните
4. На главном экране нажмите **«Обновить подписки»**
5. Выберите группу → **«Реальная задержка»** (не TCP ping!)
6. Выберите сервер с наименьшим числом (зелёный) → ▶️

### Android — Karing

1. Скачайте [Karing](https://github.com/KaringX/karing/releases) с GitHub
2. Добавить профиль → «Добавление подписки» → вставьте ссылку
3. Интервал обновления: 1 час
4. Настройки → «Автовыбор» → включить
5. Нажмите щит 🛡️ — клиент сам выберет лучший сервер

### Android — NekoBox

1. Скачайте [NekoBox](https://github.com/MatsuriDayo/NekoBoxForAndroid/releases)
2. Профили → «Добавить» → «URL» → вставьте ссылку
3. Нажмите на профиль → «Обновить» → выберите сервер → подключитесь

### iOS — Streisand (бесплатный)

1. Установите [Streisand](https://apps.apple.com/us/app/streisand/id6450534064) из App Store
2. **+** → «Импорт из буфера обмена» или введите URL подписки
3. После загрузки выберите сервер → «Подключить»

### iOS — Karing (бесплатный, рекомендуется)

1. Установите [Karing](https://apps.apple.com/us/app/karing/id6472431552) из App Store
2. Добавить профиль → «Добавление подписки» → вставьте ссылку
3. Включите автовыбор

### iOS — v2Box (бесплатный)

1. Установите [v2Box](https://apps.apple.com/us/app/v2box-v2ray-client/id6446814690) из App Store
2. Subscriptions → **+** → вставьте URL → сохраните → обновите

### Windows — v2rayN

1. Скачайте [v2rayN](https://github.com/2dust/v2rayN/releases) → распакуйте → запустите от имени администратора
2. Настройки → Региональные пресеты → «Россия» → перезапустите
3. Группа подписки → Настройки группы → **+** → вставьте ссылку
4. Группа подписки → «Обновить без прокси»
5. Выберите конфиги → правая кнопка → «Тест реальной задержки»
6. Отсортируйте по пингу → выберите лучший → Enter

### Windows — Karing

1. Скачайте [Karing](https://github.com/KaringX/karing/releases) → запустите от администратора
2. Добавить профиль → «Добавление подписки» → вставьте ссылку → интервал 1 час
3. Настройки → Автовыбор → интервал 10 мин, погрешность 150 мс
4. Нажмите щит 🛡️

## ❓ Частые вопросы

<details>
<summary><strong>Конфиги добавил, но ни один не работает. Что делать?</strong></summary>

1. Убедитесь, что используете `verified_configs.txt`, а не `all_configs.txt`
2. Нажмите «Тест реальной задержки» (не TCP ping — он не показывает реальную доступность VPN)
3. Попробуйте другой клиент — один клиент иногда «не видит» конфиги, которые видит другой
4. На мобильном интернете попробуйте `ru_configs.txt`
5. Обновите подписку вручную — конфиги обновляются каждые 6 часов

</details>

<details>
<summary><strong>Чем отличается «TCP ping» от «реальной задержки»?</strong></summary>

**TCP ping** — проверяет, открыт ли порт. Сервер может ответить, но VPN при этом не работает (порт открыт, но сервис перегружен или упал).

**Реальная задержка (Real Delay)** — клиент реально устанавливает VPN-соединение и проверяет доступность интернета через него. Зелёное число — VPN реально работает.

Всегда используйте **реальную задержку** для выбора рабочего сервера.

</details>

<details>
<summary><strong>Почему конфиги работали вчера, а сегодня нет?</strong></summary>

Публичные бесплатные сервера не имеют гарантий стабильности. Они могут быть заблокированы РКН, перегружены большим числом пользователей или отключены владельцем.

Поэтому подписки обновляются каждые 6 часов. Включите **автообновление** в клиенте (интервал 1–2 часа) — это самый надёжный способ всегда иметь рабочие конфиги.

</details>

<details>
<summary><strong>Безопасно ли использовать публичные VPN-конфиги?</strong></summary>

Публичные конфиги — это бесплатные серверы от незнакомых операторов. Основные риски: владелец сервера теоретически видит нешифрованный трафик, который вы через него пропускаете, и нет гарантий отсутствия логирования.

**Рекомендация:** не передавайте через публичные конфиги пароли и банковские данные. Для повышенной приватности используйте личный ключ на выделенном сервере.

</details>

<details>
<summary><strong>Что делать, если GitHub заблокирован?</strong></summary>

Используйте зеркала — они работают даже при заблокированном GitHub:

| Зеркало | Как использовать |
|---------|-------------------|
| **GitHack** (рекомендуется) | Замените `raw.githubusercontent.com` на `raw.githack.com` в любой ссылке выше |
| **jsDelivr** | `https://cdn.jsdelivr.net/gh/aviamastersgh/vpn-free-russia@main/verified_configs.txt` |

</details>

<details>
<summary><strong>Можно ли использовать конфиги на роутере или нескольких устройствах?</strong></summary>

Да. Клиенты **v2rayN** (Windows) и **Karing** (все платформы) поддерживают режим TUN, направляющий весь трафик устройства через VPN. Для роутеров подойдут клиенты с поддержкой Mihomo (Clash Meta) — например, OpenClash для OpenWrt.

</details>

<details>
<summary><strong>Какой протокол выбрать: VLESS, Shadowsocks или Hysteria2?</strong></summary>

Для максимальной устойчивости к блокировкам РКН — **VLESS + Reality**: маскирует трафик под обращение к реальному HTTPS-сайту, что затрудняет распознавание DPI. **Hysteria2** хорош по скорости на UDP-based соединениях. **Shadowsocks** и **VMess** — более старые протоколы, но остаются рабочими и широко поддерживаются клиентами.

</details>
