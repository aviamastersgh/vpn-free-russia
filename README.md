## 📂 Три файла — в чём разница?

Это главный вопрос новичков. Вот простое объяснение:

### `all_configs.txt` — Все найденные конфиги

Содержит **все** конфиги, найденные во всех источниках, после удаления дубликатов. Часть из них может не работать прямо сейчас — сервера бывают перегружены, отключены или заблокированы. Используйте этот файл если хотите максимальный выбор и сами готовы проверить что работает.

➡️ **Когда брать:** у вас есть время самому протестировать, или стандартный файл не помогает.

```

```

---

### `verified_configs.txt` — Только рабочие ✅ (рекомендуется)

Из всего пула конфигов отбираются только те, к которым удалось установить **реальное TCP-соединение** в момент проверки. Это значит: сервер существует, отвечает, порт открыт. Таких конфигов меньше, но они с высокой вероятностью заработают у вас.

➡️ **Когда брать:** в большинстве случаев — берите именно этот файл. Это лучший выбор для начала.

## 🤔 Как определить, какой файл нужен мне?

**Шаг 1.** Проверьте базово: открывается ли у вас `google.com` без VPN?

- **Да, открывается** → у вас обычный интернет (чёрные списки). Берите `verified_configs.txt`.

## 📱 Как добавить конфиги в клиент — пошаговые инструкции

### Android — v2rayNG

1. Скачайте [v2rayNG](https://github.com/2dust/v2rayNG/releases) с GitHub
2. Откройте приложение → нажмите ☰ (меню) → **«Группа подписки»**
3. Нажмите **+** → вставьте ссылку на подписку → сохраните
4. На главном экране нажмите **«Обновить подписки»** (значок обновления)
5. Нажмите на группу → выберите **«Реальная задержка»** (не TCP ping!)
6. Выберите сервер с наименьшим числом (зелёный) → нажмите ▶️

### Android — Karing

1. Скачайте [Karing](https://github.com/KaringX/karing/releases) с GitHub
2. Добавить профиль → **«Добавление подписки»** → вставьте ссылку
3. Интервал обновления: 1 час
4. В настройках найдите «Автовыбор» → включите
5. Нажмите щит 🛡️ для подключения — клиент сам выберет лучший сервер

### Android — NekoBox

1. Скачайте [NekoBox](https://github.com/MatsuriDayo/NekoBoxForAndroid/releases)
2. Профили → **«Добавить»** → **«URL»** → вставьте ссылку
3. Нажмите на профиль → «Обновить» → выберите сервер → подключитесь

### iOS — Streisand (бесплатный)

1. Установите [Streisand](https://apps.apple.com/us/app/streisand/id6450534064) из App Store
2. Нажмите **+** → **«Импорт из буфера обмена»** или введите URL подписки
3. После загрузки выберите сервер → нажмите «Подключить»

### iOS — Karing (бесплатный, рекомендуется)

1. Установите [Karing](https://apps.apple.com/us/app/karing/id6472431552) из App Store
2. Добавить профиль → «Добавление подписки» → вставьте ссылку
3. Включите автовыбор — клиент сам найдёт рабочий сервер

### iOS — v2Box (бесплатный)

1. Установите [v2Box](https://apps.apple.com/us/app/v2box-v2ray-client/id6446814690) из App Store
2. Subscriptions → **+** → вставьте URL → сохраните → обновите

### Windows — v2rayN

1. Скачайте [v2rayN](https://github.com/2dust/v2rayN/releases) → распакуйте → запустите от имени администратора
2. Настройки → Региональные пресеты → **«Россия»** → перезапустите
3. Группа подписки → Настройки группы → **+** → вставьте ссылку
4. Группа подписки → **«Обновить без прокси»**
5. Выберите конфиги → правая кнопка → **«Тест реальной задержки»**
6. Отсортируйте по пингу → выберите лучший → **Enter**

### Windows — Karing

1. Скачайте [Karing](https://github.com/KaringX/karing/releases) → запустите от администратора
2. Добавить профиль → «Добавление подписки» → вставьте ссылку → интервал 1 час
3. Настройки → Автовыбор → интервал 10 мин, погрешность 150 мс

## ❓ Частые вопросы

<details>
<summary><strong>Конфиги добавил, но ни один не работает. Что делать?</strong></summary>

1. Убедитесь, что используете `verified_configs.txt`, а не `all_configs.txt`
2. Нажмите «Тест реальной задержки» (не TCP ping — он не показывает реальную доступность VPN)
3. Попробуйте другой клиент — иногда один клиент «не видит» конфиги, которые видит другой
4. Если у вас мобильный интернет — попробуйте `ru_configs.txt`
5. Обновите подписку вручную (кнопка обновления в клиенте) — конфиги обновляются каждые 6 часов

</details>

<details>
<summary><strong>Чем отличается «TCP ping» от «реальной задержки»?</strong></summary>

**TCP ping** — просто проверяет, открыт ли порт. Сервер может ответить на ping, но VPN при этом не работает (например, порт открыт, но сервис упал или перегружен).

**Реальная задержка** — клиент реально устанавливает VPN-соединение и проверяет доступность интернета через него. Если зелёное число — VPN реально работает.

Всегда используйте **реальную задержку** для выбора рабочего сервера.

</details>

<details>
<summary><strong>Почему конфиги работали вчера, а сегодня нет?</strong></summary>

Публичные бесплатные сервера не имеют гарантий стабильности. Они могут:
- Быть заблокированы РКН
- Перегрузиться от большого числа пользователей
- Быть отключены владельцем

Поэтому подписки обновляются каждые 6 часов. Включите **автообновление** в вашем клиенте (интервал 1–2 часа) — и вы всегда будете получать актуальные рабочие конфиги автоматически.

</details>

<details>
<summary><strong>Безопасно ли использовать публичные конфиги?</strong></summary>

Публичные конфиги — это бесплатные серверы, предоставляемые незнакомыми людьми. Основные риски:
- Владелец сервера теоретически видит незашифрованный трафик, который вы через него пропускаете
- Нет гарантий логирования (или отсутствия логирования)

**Рекомендации:** не передавайте через публичные конфиги пароли, банковские данные и другую чувствительную информацию. Для безопасного использования — берите личный ключ в [@NosokVPNBot](https://t.me/NosokVPNBot?start=partner_8655864538).
